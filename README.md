# Mrarviher - 现代化文本编辑器

一个简洁、高效的跨平台文本编辑器，专注于纯文本编辑体验，而非IDE功能。

## 核心理念

**Mrarviher 是文本编辑器，不是 IDE。**

我相信文本编辑器和集成开发环境（IDE）应该有明确的界限：

| 特性 | Mrarviher（文本编辑器） | IDE（如VS Code） |
|------|------------------------|------------------|
| **定位** | 轻量级文本编辑 | 全功能开发环境 |
| **启动速度** | 即时启动 | 较慢加载 |
| **内存占用** | 极低 | 较高 |
| **语法高亮** | 仅预览区（Markdown代码块） | 编辑器实时高亮 |
| **智能提示 | 无 | 有 |
| **调试功能 | 无 | 完整调试器 |
| **项目管理 | 无 | 工作区/项目支持 |
| **终端集成 | 无 | 内置终端 |
| **插件系统 | 无 | 丰富插件生态 |

## 为什么不是IDE？

### Notepad++ 的问题
Notepad++ 从一个优秀的文本编辑器逐渐演变成了"轻量级IDE"：
- 添加了语法高亮
- 变得越来越复杂

### Mrarviher 的选择
我选择**回归本质**：
- **专注编辑**：打开文件，编辑内容，保存关闭
- **无干扰界面**：没有复杂的侧边栏、面板、状态栏堆叠
- **快速响应**：不加载不需要的功能模块
- **简单可靠**：减少出错可能，提升稳定性

## 主要功能

### 文件支持
- **全格式兼容**：支持所有非二进制文本文件
  - Markdown (.md, .markdown)
  - HTML (.html, .htm, .xhtml)
  - 纯文本 (.txt)
  - 配置文件 (.json, .xml, .yaml, .ini, .cfg)
  - 代码文件 (.js, .py, .c, .cpp, .java, .css等)
- **智能检测**：自动识别二进制文件并拒绝打开
- **文件关联**：可注册为默认文本编辑器

### 编辑模式
- **Markdown模式**：分屏预览 + 实时渲染
- **HTML模式**：直接渲染HTML内容到预览区
- **纯文本模式**：仅编辑器，简洁无干扰

### 自定义选项
- **主题色**：8种预设颜色可选
- **深色/浅色模式**：全局应用主题切换
- **字体设置**：
  - 系统字体自动检测
  - 6种推荐字体
  - 自定义字体大小（编辑器/预览独立设置）
- **编辑器配置**：
  - Tab缩进大小（2/4空格）
  - Tab输出制表符或空格
  - 自动换行开关
- **自动保存**：可选的延迟自动保存功能

### 用户体验
- **自定义标题栏**：现代化窗口控制
- **格式化工具栏**：快捷插入Markdown语法（加粗、斜体、标题等）
- **视图切换**：分屏 / 仅编辑 / 仅预览
- **拖拽分割线**：自由调整编辑器和预览区比例
- **未保存提示**：防止意外关闭丢失数据
- **恢复上次文件**：启动时自动打开上次编辑的文件

## 技术栈

- **框架**：Electron + Node.js
- **前端**：原生 HTML/CSS/JavaScript
- **Markdown解析**：marked.js (GFM标准)
- **代码高亮**：highlight.js (仅预览区)
- **包管理**：pnpm
- **构建工具**：electron-builder

## 适用场景

### 推荐使用
- ✅ 快速查看和编辑文本文件
- ✅ 编写Markdown文档并实时预览
- ✅ 编辑配置文件（.json, .yaml等）
- ✅ 查看HTML页面效果
- ✅ 作为系统默认文本编辑器替代记事本
- ✅ 快速记录笔记和想法

### 不适合使用
- ❌ 大型项目开发（请用IDE）
- ❌ 需要代码调试（请用IDE）
- ❌ 需要Git图形化界面（请用专用工具）
- ❌ 需要远程开发（请用VS Code Remote）
- ❌ 需要数据库管理（请用专业工具）

## 安装与构建

### 重要说明：无预编译安装包

**由于技术限制，我们无法直接提供预编译的安装程序（.exe安装包）。**

原因：
- Windows安装包需要代码签名证书（需付费购买）
- 自动修改注册表需要管理员权限提升
- electron-builder在CI/CD环境下的签名流程复杂

**解决方案：使用自动化构建脚本**

我们提供了 `setup.bat` 脚本，可一键完成从零到运行的全部步骤。

### 快速开始（推荐）

#### 方法一：自动构建脚本

双击运行 `setup.bat`，脚本会自动完成以下操作：

1. **检测/下载 Node.js**
   - 如未安装Node.js，自动从国内镜像源下载
   - 使用 npmmirror 镜像加速下载
   - 静默安装，无需手动操作

2. **安装 pnpm**
   - 全局安装 pnpm 包管理器
   - 如已存在则跳过

3. **安装项目依赖**
   - 执行 `pnpm install` 安装所有依赖

4. **构建应用**
   - 执行 `pnpm build` 打包应用
   - 输出到 `dist/win-unpacked/` 目录

5. **文件关联设置（手动）**
   - 构建完成后，提示用户手动执行注册表文件
   - 双击 `install-file-association.reg`
   - 在弹出的确认框中点击"是"

**完整命令行操作：**
```bash
# 双击 setup.bat 或在命令行执行：
setup.bat
```

#### 方法二：手动构建（开发者）

如果你已熟悉 Node.js 环境：

```bash
# 前置要求
# 1. 安装 Node.js: https://nodejs.org/
# 2. 安装 pnpm: npm install -g pnpm

# 克隆项目后：
cd Mrarviher-ElectroNode

# 安装依赖
pnpm install

# 开发模式运行
pnpm start

# 打包应用
pnpm build

# 签名打包（Windows，需要证书）
pnpm build:signed
```

### 文件关联设置

构建完成后，如需将 Mrarviher 注册为默认文本编辑器：

1. 找到项目根目录下的 `install-file-association.reg` 文件
2. 双击该文件
3. 在弹出的注册表编辑器确认框中点击"是"
4. 完成后，双击 .md 文件会自动用 Mrarviher 打开

**注意：** 修改注册表需要管理员权限。
